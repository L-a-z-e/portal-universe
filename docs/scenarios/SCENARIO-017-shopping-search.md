# SCENARIO-017: 상품 검색 및 검색어 관리

## Overview

사용자가 **키워드로 상품을 검색**하고, **자동완성 제안**을 활용하며, **인기 검색어**와 **최근 검색어**를 관리하는 E2E 시나리오입니다.

---

## Actors

| Actor | 역할 | 책임 |
|-------|------|------|
| **사용자** | 검색 수행자 | 키워드 입력, 검색 실행 |
| **Shopping Service** | 검색 처리 | 상품 검색, 검색어 추천 |

---

## Preconditions

- 상품이 1개 이상 등록된 상태
- 사용자가 로그인된 상태 (최근 검색어용)

---

## Flow

### 메인 시나리오: 검색 → 결과 확인

```
1. 사용자가 상품 목록 페이지(/) 접속
2. 검색 입력 필드에 키워드 입력
3. 자동완성 드롭다운에 제안 검색어 표시
4. 제안 검색어 클릭 또는 Enter 키로 검색 실행
5. 검색 결과 상품 목록 표시
6. 최근 검색어에 해당 키워드 자동 추가
```

### 대안 시나리오: 인기 검색어 활용

```
1. 검색 필드 포커스 시 인기 검색어 표시
2. 인기 검색어 클릭으로 검색 실행
3. 해당 키워드 검색 결과 표시
```

### 대안 시나리오: 최근 검색어 관리

```
1. 검색 영역에서 최근 검색어 목록 표시
2. 최근 검색어 개별 삭제
3. 최근 검색어 전체 삭제
4. 최근 검색어 클릭으로 재검색
```

### 대안 시나리오: 검색 결과 없음

```
1. 존재하지 않는 키워드 검색
2. "검색 결과가 없습니다" 빈 상태 표시
```

## API Endpoints

| HTTP | Endpoint | 설명 |
|------|----------|------|
| GET | `/search/products` | 상품 검색 (키워드, 가격, 정렬, 페이징) |
| GET | `/search/suggest` | 자동완성 제안 |
| GET | `/search/popular` | 인기 검색어 |
| GET | `/search/recent` | 최근 검색어 조회 |
| POST | `/search/recent` | 최근 검색어 추가 |
| DELETE | `/search/recent/{keyword}` | 최근 검색어 삭제 |
| DELETE | `/search/recent` | 최근 검색어 전체 삭제 |
