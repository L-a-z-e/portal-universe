## 학습 목표

- [ ] Post-Mortem의 목적을 이해한다
- [ ] 비난 없는(Blameless) Post-Mortem 문화를 이해한다

---

## 1. Post-Mortem이란?

### 목적

> 같은 장애가 다시 발생하지 않도록 **학습**하고 **개선**하는 것

### 원칙: Blameless

- ❌ "누가 잘못했나?"
- ✅ "시스템/프로세스가 왜 이것을 허용했나?"

## 3. Post-Mortem 프로세스

### 일정

1. **24시간 내**: 초안 작성
2. **72시간 내**: 팀 검토 미팅
3. **1주일 내**: 액션 아이템 시작
4. **2주 후**: 진행 상황 점검

### 검토 미팅

- **참석자**: 대응 참여자 + 관련 팀
- **시간**: 30-60분
- **목표**: 타임라인 확인, 액션 아이템 결정

---

## 4. 액션 아이템 유형

| 유형 | 예시 |
|------|------|
| **예방** | 이중화 구성, 입력 검증 추가 |
| **감지** | 알림 규칙 추가, 대시보드 개선 |
| **대응** | Runbook 업데이트, 자동화 |
| **완화** | Circuit Breaker 설정 조정 |
