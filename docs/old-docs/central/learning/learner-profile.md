## 💡 학습 인사이트

> 학습하면서 깨달은 점, 연결고리, 아하 모먼트 기록

- **2026-01-20**: synchronized는 자동문, ReentrantLock은 수동문이라는 비유로 차이점을 명확히 이해함
- **2026-01-20**: new Object()가 Lock이 될 수 있는 이유는 모든 Java 객체가 Mark Word를 가지기 때문!
- **2026-01-20**: 데드락 방지의 핵심은 "모든 스레드가 같은 순서로 Lock을 획득하게 하는 것"

## 📌 Profile 관리 규칙

### 자동 업데이트 시점
1. **새 영역 질문 시**: 해당 카테고리에 새 항목 추가 (레벨 1)
2. **세션 종료 시**: 학습한 내용 기반 레벨 조정
3. **반복 질문 시**: 반복 질문 영역에 추가
4. **이해 확인 시**: 레벨 상향 조정

### 레벨 조정 기준
- **올림**: 코드 보고 바로 이해 / 직접 작성 가능 / 퀴즈 통과
- **내림**: 같은 질문 반복 / 다시 봤을 때 이해 안 됨

### 수동 명령어
```bash
/learn --profile          # 현재 프로필 확인
/learn --profile update   # 세션 종료 시 업데이트 요청
```
